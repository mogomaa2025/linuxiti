File & Directory Listing
ls [options] [directory]
List files and directories with various formatting options
ls -a # Show hidden files
ls -l # Detailed list format

Directory Navigation

Change directories and display current working directory

Directory Management

Create and remove directories with various options

rmdir newdir # Remove empty directory
📋
Copy & Move
cp [options] source dest
Copy and move files or directories
cp file1 newdir/ # Copy file
cp -r dir1 dir2 # Copy directory
mv file1 newdir/ # Move file
📖
File Reading

Read file contents in different ways

head -n5 file1 # First 5 lines
tail -n2 file1 # Last 2 lines
🔢
Word Count

Count lines, words, and characters in files
wc file1 # Lines, words, bytes
wc -l file1 # Line count only

⚡ Efficient User & Group Management

While understanding files like /etc/passwd is crucial, daily user management is done with dedicated commands for efficiency and safety.

✨
Add User (useradd)
sudo useradd [options] LOGIN
Creates a new user. By default, it creates a home directory and a group with the same name, but leaves the account locked until a password is set.
sudo useradd newuser
# Check /etc/shadow, you'll see '!' meaning locked
sudo cat /etc/shadow | grep newuser # newuser:!:...
🔑
Set Password (passwd)
sudo passwd USERNAME
The secure, interactive way to set or change a user's password, which unlocks the account.
sudo passwd newuser # Follow prompts
🔧
Modify User (usermod)
sudo usermod [options] LOGIN
Modify an existing user's attributes, such as groups, comment/full name, or shell. Use -aG to append to groups.
sudo usermod -aG wheel newuser # Add user to wheel group
sudo usermod -c "New User" newuser # Change comment
sudo usermod -L newuser # Lock the account
sudo usermod -U newuser # Unlock the account
❌
Remove User (userdel)
sudo userdel [options] LOGIN
Deletes a user account. The -r flag is crucial to also remove the user's home directory.
sudo userdel -r newuser # Removes user and their data
➕
Add Group (groupadd)
sudo groupadd [options] GROUP
Creates a new group. You can add existing users to the new group upon creation.
sudo groupadd developers # Create a new group
sudo groupadd -U gomaa newgroup # Create group and add user
🆔
Check User ID (id)
id [USERNAME]
Displays comprehensive user and group ID information (uid, gid, and all group memberships) for a specified user.
id gomaa
# uid=1000(gomaa) gid=1000(gomaa) groups=...

🔒 Security Concepts

📊

Encoding vs Encryption vs Hashing

🔄 Encoding: Data ➔ String (reversible)
Purpose: Data representation, not security
🔐 Encryption: Data + Key ➔ Encrypted String (reversible with key)
Purpose: Confidentiality, data protection
#️⃣ Hashing: Data ➔ Hash (irreversible, one-way)
Purpose: Integrity, password storage, git commits
🧂

Password Security & Salting

Hash Storage: Passwords never stored in plain text
System stores hash, compares hash of input
Salting Protection: Random data + password before hashing
Prevents rainbow table and brute-force attacks
Modern Algorithm: $y$ prefix = yescrypt (Argon2-based)
Strong, slow hashing designed for security

💡 Pro Tips

🔗 Command Chaining

Use ; to run multiple commands sequentially: ls ; whoami

👁️ Hidden Files

Files starting with . are hidden. Use ls -a to see them all, or ls -A to exclude . and ..

🛣️ Path Types

Relative: cd Desktop | Absolute: cd /home/user/Desktop

📁 Directory Operations

Use mkdir -p to create nested directories in one command

🔐 System Account Security

Accounts with * or ! in /etc/shadow are locked and cannot log in - good security practice for system services.

🏠 Home Directory Notes

Some system users have no home directory or use / - they don't need interactive access

🧪 Comprehensive Lab Exercise

Apply your knowledge! Here are the step-by-step solutions for the lab exercise. (Assuming your username is 'user' and you start in /home/user).

1. Create Directory Structure

mkdir -p dir1/dir11 dir1/dir12 doc
touch dir1/dir11/file1 doc/file2

mkdir -p creates parent directories as needed. touch creates the empty files in their respective locations.

2. Copy /etc/passwd

cp /etc/passwd ~/mypasswd

3. Rename mypasswd

mv ~/mypasswd ~/oldpasswd

4. Write Absolute Path for file2

The full path from the root (/) is: /home/user/doc/file2

5. Write Relative Path for file2

From /home/user, the relative path is: doc/file2

6. Four Ways to Go Home from /usr/bin

cd
cd ~
cd $HOME
cd /home/user

7. List All Hidden Files

ls -a ~

8. Copy dir1 to Downloads as dir4

cp -r ~/dir1 ~/Downloads/dir4

The -r flag is required to copy a directory and its contents recursively.

9. Display First 4 Lines of /etc/passwd

head -n 4 /etc/passwd

10. Display Last 7 Lines of /etc/passwd

tail -n 7 /etc/passwd

11. Remove dir11 in One Step

rm -r dir1/dir11

You'd notice that rmdir fails because dir11 contains file1. You must use rm -r to remove a non-empty directory.

12. Count Words in /etc/passwd

wc -w /etc/passwd

13. Count Lines in /etc/passwd

wc -l /etc/passwd

14. Create User 'islam'

sudo useradd -c "Islam ITI" islam
sudo passwd islam

First, create the user with a comment. Second, set the password interactively. You will be prompted to enter 'islam'.

15. Create User 'baduser'

sudo useradd -c "Bad User" baduser
sudo passwd baduser

Follow the same two-step process to create the user and set their password.

16. Create Group 'pgroup' with GID 30000

sudo groupadd -g 30000 pgroup

17. Create Group 'badgroup'

sudo groupadd badgroup

18. Add User 'islam' to 'pgroup'

sudo usermod -aG pgroup islam

-a appends the user to the group, and -G specifies the supplementary group(s).

19. Modify Password for 'islam'

sudo passwd islam

You will be prompted to enter the new password, 'password'.

20. Lock the 'baduser' Account

sudo usermod -L baduser

21. Delete the 'baduser' Account

sudo userdel -r baduser

The -r flag ensures the user's home directory is also removed.

22. Delete the Group 'badgroup'

sudo groupdel badgroup
