🐧 Linux Bash Shell

Master the command line with this comprehensive guide to essential bash commands

📁
File & Directory Listing
ls [options] [directory]
List files and directories with various formatting options
ls -a # Show hidden files
ls -l # Detailed list format
ls -A # All files except . and ..
📍
Directory Navigation
cd [directory] && pwd
Change directories and display current working directory
cd Desktop # Relative path
cd /home/user # Absolute path
pwd # Show current directory
🔍
File Operations
file [filename]
Determine file type and get file information
file Desktop/ # Check if directory
file document.txt # Check file type
👤
User Information
whoami && who
Display current user and active login sessions
whoami # Current username
who # All logged in users
📂
Directory Management
mkdir [options] directory
Create and remove directories with various options
mkdir newdir # Create directory
mkdir -p dir1/dir2 # Create hierarchy
rmdir newdir # Remove empty directory
📄
File Creation & Editing
touch filename
Create files or update timestamps
touch newfile # Create empty file
touch existing.txt # Update timestamp
📋
Copy & Move
cp [options] source dest
Copy and move files or directories
cp file1 newdir/ # Copy file
cp -r dir1 dir2 # Copy directory
mv file1 newdir/ # Move file
🗑️
File Removal
rm [options] files
Remove files and directories safely
rm file1 # Remove file
rm -r directory # Remove directory
rm -rf directory # Force remove
📖
File Reading
cat | head | tail filename
Read file contents in different ways
cat file1 # Read entire file
head -n5 file1 # First 5 lines
tail -n2 file1 # Last 2 lines
🔢
Word Count
wc [options] filename
Count lines, words, and characters in files
wc file1 # Lines, words, bytes
wc -l file1 # Line count only
wc -w file1 # Word count only
📜
File Paging
more | less filename
View large files page by page
more /etc/passwd # Page through file
less file1 # Navigate with arrows
🕐
Date & Time
date
Display current date and time
date # Current date and time

🗂️ Linux File System

/
Root directory - the top of the file system hierarchy
/boot
Boot files required to start the operating system
/etc
System configuration files (firewall, users, ports)
/home
User home directories (like /home/gomaa)
/root
Root user's home directory
/usr
User programs and system binaries
/var
Variable data like logs and databases
/tmp
Temporary files (cleared on reboot)
/dev
Device files (hardware interfaces)
/mnt
Mount point for temporary mounted filesystems

🧪 Practical Lab Walkthrough

Apply your knowledge! Here are the step-by-step solutions for a common Linux command-line exercise. (Assuming your username is 'user' and you are starting in your home directory /home/user).

Question 2: Create the Directory Hierarchy

To create the directories (including nested ones) and the empty files, we use mkdir -p and touch.

mkdir -p dir1/dir11 dir1/dir12 doc
touch dir1/dir11/file1 doc/file2

The -p flag creates parent directories as needed, so dir1 is made automatically when creating its subdirectories.

Question 3: Copy /etc/passwd

Use the cp command. The source is /etc/passwd and the destination is your home directory (~) with the new name mypasswd.

cp /etc/passwd ~/mypasswd

Question 4: Rename mypasswd to oldpasswd

The mv (move) command is used for both moving and renaming files.

mv ~/mypasswd ~/oldpasswd

Question 5: Find Absolute and Relative Paths

Given your current directory is /home/user, determine the paths to file2.

Absolute Path: The full path from the root (/).
/home/user/doc/file2

Relative Path: The path from your current location.
doc/file2

Question 6: Four Ways to Return Home

If you are in /usr/bin, there are several shortcuts to get back to your home directory.

cd
cd ~
cd $HOME
cd /home/user # (Using the explicit absolute path)

Question 7: List All Hidden Files

Use ls with the -a flag to show all entries, including those starting with a dot (.).

ls -a ~

Use ls -A if you want to see hidden files but exclude the . (current) and .. (parent) directory entries.

Question 8: Copy a Directory

To copy a directory and its contents, use cp with the -r (recursive) flag. This command copies dir1 to your Downloads folder and renames it to dir4.

cp -r ~/dir1 ~/Downloads/dir4

Note: This assumes a ~/Downloads directory exists. If not, you would create it first with mkdir ~/Downloads.

👥 User Management & Security

👤
User Information Files
cat /etc/passwd
View all user accounts and their configuration details
# Format: loginname:x:UID:GID:comment:home:shell
root:x:0:0:Super User:/root:/bin/bash
gomaa:x:1000:1000:gomaa:/home/gomaa:/bin/bash
🔐
Password Shadows
sudo cat /etc/shadow
View encrypted password hashes and account policies (requires sudo)
# Passwords stored as irreversible hashes
gomaa:$y$j9T$hash...
bin:*:20186:0:99999:7::: # Locked account
👥
Group Management
cat /etc/group
View all groups and their members
# Format: groupname:x:GID:members
wheel:x:10:gomaa # Admin group
root:x:0: # Root group

⚡ Efficient User & Group Management

While understanding files like /etc/passwd is crucial, daily user management is done with dedicated commands for efficiency and safety.

✨
Add User (useradd)
sudo useradd [options] LOGIN
Creates a new user. By default, it creates a home directory and a group with the same name, but leaves the account locked until a password is set.
sudo useradd newuser
# Check /etc/shadow, you'll see '!' meaning locked
sudo cat /etc/shadow | grep newuser # newuser:!:...
🔑
Set Password (passwd)
sudo passwd USERNAME
The secure, interactive way to set or change a user's password, which unlocks the account.
sudo passwd newuser # Follow prompts
🔧
Modify User (usermod)
sudo usermod [options] LOGIN
Modify an existing user's attributes, such as groups, comment/full name, or shell. Use -aG to append to groups.
sudo usermod -aG wheel newuser # Add user to wheel group
sudo usermod -c "New User" newuser # Change comment
sudo usermod -L newuser # Lock the account
sudo usermod -U newuser # Unlock the account
❌
Remove User (userdel)
sudo userdel [options] LOGIN
Deletes a user account. The -r flag is crucial to also remove the user's home directory.
sudo userdel -r newuser # Removes user and their data
➕
Add Group (groupadd)
sudo groupadd [options] GROUP
Creates a new group. You can add existing users to the new group upon creation.
sudo groupadd developers # Create a new group
sudo groupadd -U gomaa newgroup # Create group and add user
🆔
Check User ID (id)
id [USERNAME]
Displays comprehensive user and group ID information (uid, gid, and all group memberships) for a specified user.
id gomaa
# uid=1000(gomaa) gid=1000(gomaa) groups=...

🔒 Security Concepts

📊

Encoding vs Encryption vs Hashing

🔄 Encoding: Data ➔ String (reversible)
Purpose: Data representation, not security
🔐 Encryption: Data + Key ➔ Encrypted String (reversible with key)
Purpose: Confidentiality, data protection
#️⃣ Hashing: Data ➔ Hash (irreversible, one-way)
Purpose: Integrity, password storage, git commits
🧂

Password Security & Salting

Hash Storage: Passwords never stored in plain text
System stores hash, compares hash of input
Salting Protection: Random data + password before hashing
Prevents rainbow table and brute-force attacks
Modern Algorithm: $y$ prefix = yescrypt (Argon2-based)
Strong, slow hashing designed for security

💡 Pro Tips

🔗 Command Chaining

Use ; to run multiple commands sequentially: ls ; whoami

👁️ Hidden Files

Files starting with . are hidden. Use ls -a to see them all, or ls -A to exclude . and ..

🛣️ Path Types

Relative: cd Desktop | Absolute: cd /home/user/Desktop

🔄 File Timestamp Tricks

touch can modify file timestamps without changing content - be aware of security implications!

📁 Directory Operations

Use mkdir -p to create nested directories in one command

🔐 System Account Security

Accounts with * or ! in /etc/shadow are locked and cannot log in - good security practice for system services.

🏠 Home Directory Notes

Some system users have no home directory or use / - they don't need interactive access
