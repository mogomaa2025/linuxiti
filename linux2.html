üß™ Practical Lab Walkthrough

Apply your knowledge! Here are the step-by-step solutions for a common Linux command-line exercise. (Assuming your username is 'user' and you are starting in your home directory /home/user).

Question 2: Create the Directory Hierarchy

To create the directories (including nested ones) and the empty files, we use mkdir -p and touch.

mkdir -p dir1/dir11 dir1/dir12 doc
touch dir1/dir11/file1 doc/file2

The -p flag creates parent directories as needed, so dir1 is made automatically when creating its subdirectories.

Question 3: Copy /etc/passwd

Use the cp command. The source is /etc/passwd and the destination is your home directory (~) with the new name mypasswd.

cp /etc/passwd ~/mypasswd

Question 4: Rename mypasswd to oldpasswd

The mv (move) command is used for both moving and renaming files.

mv ~/mypasswd ~/oldpasswd

Question 5: Find Absolute and Relative Paths

Given your current directory is /home/user, determine the paths to file2.

Absolute Path: The full path from the root (/).
/home/user/doc/file2

Relative Path: The path from your current location.
doc/file2

Question 6: Four Ways to Return Home

If you are in /usr/bin, there are several shortcuts to get back to your home directory.

cd
cd ~
cd $HOME
cd /home/user # (Using the explicit absolute path)

Question 7: List All Hidden Files

Use ls with the -a flag to show all entries, including those starting with a dot (.).

ls -a ~

Use ls -A if you want to see hidden files but exclude the . (current) and .. (parent) directory entries.

Question 8: Copy a Directory

To copy a directory and its contents, use cp with the -r (recursive) flag. This command copies dir1 to your Downloads folder and renames it to dir4.

cp -r ~/dir1 ~/Downloads/dir4

Note: This assumes a ~/Downloads directory exists. If not, you would create it first with mkdir ~/Downloads.

‚ö° Efficient User & Group Management

While understanding files like /etc/passwd is crucial, daily user management is done with dedicated commands for efficiency and safety.

‚ú®
Add User (useradd)
sudo useradd [options] LOGIN
Creates a new user. By default, it creates a home directory and a group with the same name, but leaves the account locked until a password is set.
sudo useradd newuser
# Check /etc/shadow, you'll see '!' meaning locked
sudo cat /etc/shadow | grep newuser # newuser:!:...
üîë
Set Password (passwd)
sudo passwd USERNAME
The secure, interactive way to set or change a user's password, which unlocks the account.
sudo passwd newuser # Follow prompts
üîß
Modify User (usermod)
sudo usermod [options] LOGIN
Modify an existing user's attributes, such as groups, comment/full name, or shell. Use -aG to append to groups.
sudo usermod -aG wheel newuser # Add user to wheel group
sudo usermod -c "New User" newuser # Change comment
sudo usermod -L newuser # Lock the account
sudo usermod -U newuser # Unlock the account
‚ùå
Remove User (userdel)
sudo userdel [options] LOGIN
Deletes a user account. The -r flag is crucial to also remove the user's home directory.
sudo userdel -r newuser # Removes user and their data
‚ûï
Add Group (groupadd)
sudo groupadd [options] GROUP
Creates a new group. You can add existing users to the new group upon creation.
sudo groupadd developers # Create a new group
sudo groupadd -U gomaa newgroup # Create group and add user
üÜî
Check User ID (id)
id [USERNAME]
Displays comprehensive user and group ID information (uid, gid, and all group memberships) for a specified user.

üîí Security Concepts

üìä

Encoding vs Encryption vs Hashing

üîÑ Encoding: Data ‚ûî String (reversible)
Purpose: Data representation, not security
üîê Encryption: Data + Key ‚ûî Encrypted String (reversible with key)
Purpose: Confidentiality, data protection
#Ô∏è‚É£ Hashing: Data ‚ûî Hash (irreversible, one-way)
Purpose: Integrity, password storage, git commits
üßÇ

Password Security & Salting

Hash Storage: Passwords never stored in plain text
System stores hash, compares hash of input
Salting Protection: Random data + password before hashing
Prevents rainbow table and brute-force attacks
Modern Algorithm: $y$ prefix = yescrypt (Argon2-based)
Strong, slow hashing designed for security
